# 博客评论系统切换指南

## 当前评论系统状态

目前博客已配置了两种评论系统：

1. **Waline** (当前默认使用)
   - 优点：功能丰富、界面美观、支持匿名评论
   - 缺点：需要维护Vercel服务器

2. **Giscus** (备选方案)
   - 优点：基于GitHub Discussions、零服务器成本、数据透明
   - 缺点：需要GitHub账号登录才能评论

## 如何切换评论系统

我已经创建了一个简单的脚本，可以帮助你在两种评论系统之间快速切换。

### 前提条件

确保已安装Node.js和必要的依赖：

```bash
npm install js-yaml
```

### 切换到Waline

```bash
node switch-comments.js waline
```

### 切换到Giscus

```bash
node switch-comments.js giscus
```

### 重新生成博客

切换评论系统后，需要重新生成博客：

```bash
hexo clean && hexo generate && hexo server
```

## 评论系统配置文件

两种评论系统的配置都保存在 `_config.butterfly.yml` 文件中：

```yaml
# 评论系统配置
comments:
  use: Waline  # 可选：Waline, Giscus
  text: true
  lazyload: false
  count: true
  card_post_count: true

# Waline 评论配置
waline:
  serverURL: https://myblogwalinecomments.vercel.app/
  # ... 其他Waline配置 ...

# Giscus 评论配置
giscus:
  repo: swkajnBruceee/swkajnBruceee.github.io
  repo_id: R_kgDOM4d3kA
  category_id: DIC_kwDOM4d3kM4Ci4EW
  # ... 其他Giscus配置 ...
```

## 样式定制

两种评论系统都已经进行了样式定制，使它们的外观保持一致：

- Waline样式：`source/css/waline-anzhiyu.css`
- Giscus样式：`source/css/giscus-waline-style.css`

这些样式文件已经在主题配置中引入，切换评论系统时无需修改。

## 故障排除

### Waline评论系统问题

如果Waline评论系统无法加载：

1. 检查Vercel服务是否正常运行
2. 确认`serverURL`配置是否正确
3. 查看浏览器控制台是否有错误信息

### Giscus评论系统问题

如果Giscus评论系统无法加载：

1. 确保GitHub仓库已启用Discussions功能
2. 检查`repo`和`repo_id`配置是否正确
3. 验证`category_id`是否有效
4. 确认已安装Giscus应用到仓库
